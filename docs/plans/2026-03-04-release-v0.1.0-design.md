# HA NOVA v0.1.0 Release Design

**Date:** 2026-03-04
**Status:** Approved
**Platform:** macOS-first
**Distribution:** `npx ha-nova`

## Summary

Ship HA NOVA v0.1.0 as a macOS-first release via npm. Adds debugging (traces) and service calls to complete the MVP workflow, cleans leaked secrets from git history, removes internal dev docs, and publishes to npm + GitHub Releases.

## Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Platform | macOS-first | Ship fast; Linux/Windows follow later |
| Distribution | `npx ha-nova` | One command, npm ecosystem |
| Git cleanup | BFG Repo Cleaner | Remove `.env.local` from all history |
| Internal docs | Delete | breadcrumbs, choices, plans/, phase-*, research/ |
| New features | Traces + Service Calls | Complete the "create → debug → control" workflow |

## Current Capabilities (v0.0.x)

- Automation CRUD (create, read, update, delete) — tested
- Script CRUD (create, read, update, delete) — tested
- Entity Discovery (search/resolve by name/domain) — tested
- State Inspection (entity states via REST proxy) — tested
- 3-phase write flow (resolve → preview → apply) — tested
- Read-back verification after writes — tested
- Token-based delete confirmation — tested
- Best-practice gate (tiered: simple/complex) — tested

## New Features for v0.1.0

### A) Automation/Script Trace Debugging

**Purpose:** Answer "Why didn't my automation fire?" or "What went wrong in the last run?"

**Implementation:**
- Extend `ha-nova-read` skill (or new `ha-nova-debug` intent)
- WS call `trace/list` → list recent traces for an automation/script
- WS call `trace/get` → get detailed trace (trigger evaluation, condition results, action outcomes)
- Structured output: what fired, what didn't, why
- Agent template for trace analysis

**User flow:**
```
User: "Why didn't my motion light automation trigger last night?"
HA NOVA: → resolve automation → fetch traces → analyze
→ "Last 3 runs: trigger fired at 02:14, 03:22, 05:01.
   All 3 times condition 'sun below horizon' passed but
   condition 'input_boolean.guest_mode' was ON → blocked."
```

### B) Service Calls

**Purpose:** Direct device control ("turn on the kitchen light", "set thermostat to 22°C")

**Implementation:**
- New intent in router or extend `ha-nova` skill
- REST: `POST /api/services/{domain}/{service}` with entity target
- Entity discovery (existing) → service call combination
- No token confirmation needed (reversible actions)
- Preview before execution

**User flow:**
```
User: "Turn on the living room lights"
HA NOVA: → resolve entity → preview service call
→ "Calling light.turn_on on light.living_room"
→ ✅ Done. Light is now on (brightness: 255)
```
